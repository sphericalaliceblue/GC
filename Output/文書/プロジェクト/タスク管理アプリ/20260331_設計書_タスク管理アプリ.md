# 設計書 — タスク管理アプリ（LINE WORKS連携）

**作成日：** 2026-03-31
**作成者：** 三上 隼人（開発責任者）
**バージョン：** 1.0

---

## 1. アプリ概要

LINEワークスに話しかけるだけでタスクを登録・管理できるシステム。
期限が近づくと自動でスマホに通知が届き、ボタン操作で進捗を返すと
スプレッドシートが自動更新される。

---

## 2. 全体の仕組み

```
【タスク登録】
  あなた（LINEワークス）
      ↓ 「来週金曜までに企画書作らなきゃ」
  GAS（受け取り窓口）
      ↓
  Dify（AIが内容を解析・整理）
      ↓ タスク名・期限・担当者を抽出
  GAS（スプレッドシートに保存）
      ↓
  あなた（LINEワークス）← 「登録しました！」と返信

【進捗通知】
  GAS（毎朝自動チェック）
      ↓ 期限が近いタスクを検出
  あなた（LINEワークス）← ボタン付き通知が届く
      ↓ ボタンをタップ
  GAS（スプレッドシートを自動更新）
```

---

## 3. 通知タイミングのルール

| 区分 | 期限まで | 通知するタイミング |
|------|---------|-----------------|
| 超短期 | 1日 | 当日の朝 |
| 短期 | 2〜5日 | 前日の朝 ＋ 当日の朝 |
| 中期・長期 | 6日以上 | 中間日の朝 ＋ 前日の朝 ＋ 当日の朝 |

**中間日の計算：** 登録日〜期限日の日数の半分の地点

---

## 4. スプレッドシートの構成

シート名：`タスク一覧`

| 列 | 項目名 | 内容・形式 |
|----|--------|----------|
| A | ID | 自動採番（T001, T002…） |
| B | タスク名 | 例：企画書作成 |
| C | 期限 | YYYY-MM-DD 形式 |
| D | ステータス | 未着手 / 進行中 / 順調 / 完了 |
| E | 依頼者 | 例：ユーザー |
| F | 担当者 | 例：三上 隼人 |
| G | 作成日 | YYYY-MM-DD 形式 |
| H | 最終更新日 | YYYY-MM-DD 形式（自動） |
| I | 通知予定日リスト | JSON配列（自動計算） |

---

## 5. ボタンの種類と動作

通知に届くボタン：

| ボタン | 動作 |
|--------|------|
| 1日延期 | 期限を1日後ろにずらし、通知日も再計算 |
| 3日延期 | 期限を3日後ろにずらし、通知日も再計算 |
| 5日延期 | 期限を5日後ろにずらし、通知日も再計算 |
| 順調 | ステータスを「順調」に更新 |
| 完了 | ステータスを「完了」に更新 |

---

## 6. 使用するサービスと役割

| サービス | 役割 |
|---------|------|
| GAS（Google Apps Script） | アプリ全体の司令塔。受け取り・保存・通知を担当 |
| Googleスプレッドシート | タスクデータの保管場所 |
| Dify | 「やらなきゃ」といった自然な言葉をタスク情報に変換 |
| LINEワークス Bot | スマホへの通知送信・ボタン操作の受け取り |

---

## 7. ファイル構成

```
Output/コード/
  ├── Config.gs          # 設定値（IDやキーを記入する場所）
  ├── Main.gs            # 受け取り窓口（LINEワークスからの通信を処理）
  ├── TaskManager.gs     # タスクの保存・更新・取得
  ├── LineWorksAPI.gs    # LINEワークスへのメッセージ送信
  ├── Notifier.gs        # 期限チェックと通知送信
  ├── DifyAPI.gs         # Difyとの連携
  └── セットアップガイド.md  # 設定手順書
```

---

## 8. セットアップの流れ（概要）

1. Googleスプレッドシートを作成し、シートIDをメモ
2. LINEワークス DeveloperコンソールでBotを作成し、各種IDをメモ
3. Difyでワークフローを作成し、APIキーをメモ
4. GASプロジェクトを作成し、上記コードファイルを貼り付け
5. `Config.gs` に各IDを記入
6. GASを「ウェブアプリ」として公開し、URLをLINEワークスのCallback URLに設定
7. GASのトリガーで毎朝9時に `checkDeadlines` を実行するよう設定
