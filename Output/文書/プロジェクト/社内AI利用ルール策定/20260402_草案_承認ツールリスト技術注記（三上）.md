各承認ツールのデータ管理方針・セキュリティ特性を技術的観点から整理し、規程書の付記および運用判断の根拠資料とする。

## ツール別技術注記

---

### 1. X-bot（Dify）

| 項目 | 内容 |
|------|------|
| 種別 | 社内カスタムAI（自社構築） |
| データ保存場所 | 自社管理サーバー（クラウド or オンプレミス） |
| 学習利用 | なし（自社データは外部に送信されない） |
| セキュリティ評価 | ★★★ 高（自社管理のため統制が可能） |

**注記：**
- 最もセキュリティリスクが低いツール。機密性の高い業務への活用を積極的に検討できる
- ただしDifyのバージョン管理・セキュリティパッチの適用を継続的に行うこと
- 利用ログの取得・保管設定を確認し、インシデント時のトレーサビリティを確保すること

---

### 2. ChatGPT（OpenAI）

| 項目 | 内容 |
|------|------|
| 種別 | 外部クラウドサービス |
| データ保存場所 | OpenAI社サーバー（米国） |
| 学習利用 | **ChatGPT Team/Enterprise：学習利用なし**（個人アカウントは学習利用あり） |
| セキュリティ評価 | ★★☆ 中（会社アカウントであれば学習利用なし） |

**注記：**
- 会社アカウント（Team以上のプラン）では、入力データはモデルの学習に使用されない
- ただしOpenAI社のサーバーにデータが送信される事実は変わらないため、機密情報・個人情報の入力は禁止
- 個人アカウントでの業務利用は学習利用リスクがあるため厳禁
- 入力内容はOpenAI社のプライバシーポリシーに基づき一定期間保存される点を周知すること

**規程書への記載推奨事項：**
> 「ChatGPTは会社アカウント（Teamプラン以上）のみ使用すること。個人アカウントでの業務利用は禁止。」

---

### 3. notebookLM（Google）

| 項目 | 内容 |
|------|------|
| 種別 | 外部クラウドサービス |
| データ保存場所 | Google社サーバー |
| 学習利用 | **Google Workspace契約：学習利用なし** / **個人アドレス（非Workspace）：学習利用の可能性あり** |
| セキュリティ評価 | ★☆☆ 低〜中（会社用個人アドレスのため、Workspace外の扱いとなる可能性が高い） |

**注記：**
- 現状の利用形態（会社用個人アドレス）はGoogle Workspace契約外となる可能性が高く、Googleの標準個人向け利用規約が適用される
- 標準利用規約では、入力データのサービス改善目的での利用が認められている場合がある
- **このため、notebookLMへの入力は公開情報・社外周知済み情報のみに限定すること**
- 将来的にGoogle Workspace契約への移行を検討することで、リスクを大幅に低減できる

**規程書への記載推奨事項：**
> 「notebookLMは会社用個人アドレスでの利用に限る。入力できるのは公開情報・社外周知済み情報のみとし、社内資料・機密情報・個人情報の入力を禁止する。」

---

### 4. Gamma

| 項目 | 内容 |
|------|------|
| 種別 | 外部クラウドサービス |
| データ保存場所 | Gamma社サーバー（米国） |
| 学習利用 | 有料プランでは学習利用なし（要利用規約確認） |
| セキュリティ評価 | ★★☆ 中（会社アカウント利用時） |

**注記：**
- 個人アカウントでの業務利用を廃止し、会社アカウントに統一済み（今回の決定事項）
- 主な用途はプレゼンテーション・提案資料の生成であり、入力するのは主に社内向けの内容が多い
- 社外提出用資料を作成する際は、取引先名・金額等の機密情報を除いたテキストで生成し、後から追記する運用を推奨する

**規程書への記載推奨事項：**
> 「Gammaは会社アカウントのみ使用すること。社外提出用資料の作成時は、機密情報を含まないテキストで生成し、固有情報は完成後に追記すること。」

### 5. イルシル

| 項目 | 内容 |
|------|------|
| 種別 | 外部クラウドサービス（国内） |
| データ保存場所 | 国内サーバー |
| 学習利用 | 要利用規約確認（サービスプランによる） |
| セキュリティ評価 | ★★☆ 中 |

**注記：**
- 国内サービスのため、データが国外に送信されるリスクは相対的に低い
- ただし外部サーバーへの送信という点では他のクラウドサービスと同様
- 利用規約・プライバシーポリシーの定期的な確認を推奨する

## 今後の技術評価体制について

新規ツールの申請が来た場合、DX推進事業部（および私）が以下の観点で評価する。

| 評価項目 | 確認内容 |
|---------|---------|
| データ保存場所 | 国内 or 海外 / 自社管理 or 外部 |
| 学習利用の有無 | 入力データがモデル学習に使用されるか |
| アカウント管理 | 会社単位での管理・ログ取得が可能か |
| 暗号化 | 通信・保存時のデータ暗号化の有無 |
| インシデント対応 | 情報漏洩発生時のベンダー対応方針 |

評価結果は承認ツールリストに追記し、規程書を随時更新する。
