# 現場向けAI利用ガイドブック 構成案

**作成日：** 2026年4月2日
**作成者：** 村瀬 ひとみ（人事・人材開発部長）
**ステータス：** 構成案（コンテンツ詳細はレビュー後に作成）

---

## 設計方針

| 方針 | 内容 |
|------|------|
| ターゲット | IT知識がなくても理解できること。工事現場・客先工場で働く方も含む全従業員 |
| ゴール | 「迷ったときに自分で判断できる」「困ったときに誰に聞けばよいかわかる」 |
| トーン | 禁止・罰則を前面に出さない。「使い方を知れば怖くない」スタンスで |
| 形式 | A4両面（1枚仕上げ）＋補足ページ。ラミネートして現場掲示できるレベル |

---

## ページ構成案

---

### 【表面】メインページ（最重要・必ず読む1枚）

#### ① タイトル・導入（5行以内）

```
AIって何に使っていいの？ — 3分でわかるAI利用ルール
```

難しい言葉を使わず、「このガイドを読めば安心してAIを使える」と思わせるひと言を冒頭に。

---

#### ② 使えるツール一覧（表）

```
┌──────────────┬────────────────────────────┐
│ 使っていいツール │ アカウント                  │
├──────────────┼────────────────────────────┤
│ X-bot（Dify） │ 会社アカウント              │
│ ChatGPT      │ 会社アカウント              │
│ notebookLM   │ 会社用メールアドレス        │
│ Gamma        │ 会社アカウント              │
│ イルシル      │ 会社アカウント              │
├──────────────┼────────────────────────────┤
│ それ以外       │ DX推進事業部に申請してから  │
└──────────────┴────────────────────────────┘
```

「個人のアカウントでの業務利用はNG」を一言明記。

---

#### ③ 信号機チェック（メインビジュアル）

大きく3色で表示。文字は最小限。

```
AIに入力する前に、3秒で確認！

🟢 入れていいもの
  → 会社のホームページに載っている情報
  → 世の中に公開されている情報

🟡 上司に確認してから
  → 社内の資料・手順書・連絡メール

🔴 絶対にダメ
  → お客様の名前・住所・電話番号
  → 工事現場・客先の写真・図面
  → 見積金額・契約内容
  → 社員の個人情報
```

---

#### ④ やっていいこと・ダメなことの例（2列）

| ✅ やっていいこと | ❌ やってはいけないこと |
|----------------|-------------------|
| 公開されているルールや法令を調べる | 客先の写真を送って説明を作る |
| 公開中のカタログの文章を整える | 取引先名・金額が入った書類を貼り付ける |
| メールの文章を日本語として整える（固有名詞を消してから） | AI出力をチェックせずそのまま提出する |
| 公開情報を使って勉強・調査する | 個人のアカウントで業務利用する |

---

#### ⑤ 「これってどっち？」迷ったときのひと言判断

```
「これが外に漏れたら、お客様や会社が困る？」
　→ YES なら 🔴 入力禁止
　→ NO  なら 🟢 か 🟡（上司に確認）
```

---

#### ⑥ 質問・困ったときの連絡先

```
【困ったときは一人で悩まず連絡してください】

▶ まず上司・現場責任者に確認
▶ それでもわからない場合：DX推進事業部
　 TEL：（後で記載）
　 メール：（後で記載）

※ミスしてしまったときも、すぐに報告してください。
　隠さないことが一番大事です。
```

### 【裏面】補足ページ

#### ⑦ よくある質問（Q&A形式）

現場で実際に出そうな質問を5〜7問想定して回答。
（黒木さんのリスクシナリオを参考に選定）

**想定Q&A（案）**

```
Q. 現場の写真をAIに送って報告書を作りたいんだけど？
A. 客先・工事現場の写真はNG。写真なしで文章だけ入力する方法を使ってください。

Q. お客さんの名前は出さなければいい？
A. 名前を出さなくても、状況からわかってしまう場合はNGです。迷ったら上司に確認を。

Q. ChatGPTの個人アカウントを前から使ってたんだけど？
A. 業務に使う場合は会社アカウントへの切り替えが必要です。DX推進事業部にご相談ください。

Q. AIが作った文章をそのままお客さんに送っていい？
A. 必ず自分で読んで確認してからにしてください。AIは間違えることがあります。

Q. 間違えて入力してしまったら？
A. すぐに上司に報告してください。早めの報告が大事です。罰するためではなく、対処するためです。
```

#### ⑧ 事例コーナー（怖い話ではなく学びの話として）

「AIで便利になった例」と「ヒヤリハット例」を1件ずつ。
罰則ではなく「こうすれば防げた」という学びの形式で記載。

## 研修・配布計画（案）

| 項目 | 内容 |
|------|------|
| 配布方法 | 紙（ラミネート）を現場・事務所に掲示 ＋ 社内ポータルにPDF掲載 |
| 説明タイミング | 朝礼・安全ミーティングで5分説明（既存の研修枠に組み込む） |
| 研修形式 | 集合研修ではなく、現場責任者が口頭で伝える「説明カード形式」を想定 |
| 理解度確認 | 説明後、3問のクイズ（簡易チェックシート）で確認 |
| 改訂タイミング | 承認ツールの変更・インシデント発生時に随時更新 |
