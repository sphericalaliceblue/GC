## 調査・報告内容

### 1. ターミナルでClaude Codeを使うメリット

代表からの質問を受け、以下の観点で調査・報告を実施。

| メリット | 概要 |
|---------|------|
| 開発フローへの統合 | ターミナルをベースに、編集・テスト・コミットを一貫して実行可能 |
| スクリプト化・自動化 | パイプ処理でCI/CDへの組み込みが容易 |
| ファイル・Git操作の自動化 | ファイル読み書き・Git操作が組み込みで提供 |
| CLAUDE.mdによるプロジェクト設定 | チームのルールをバージョン管理し自動読み込み |
| 非対話型（ヘッドレス）モード | 確認なしで自動処理、定期タスクにも対応 |

### 2. VS Code拡張とターミナルCLIの違い

| 比較項目 | VS Code拡張 | ターミナルCLI |
|---------|------------|-------------|
| セッション・コンテキスト | 共有 | 共有 |
| インライン差分表示 | あり | なし |
| 複数タブで並列会話 | あり | なし |
| 全コマンド・スキル利用 | 一部制限あり | すべて利用可能 |
| MCPサーバー初期設定 | 不可 | 可能 |
| Bashショートカット（`!`） | なし | あり |

**結論：** 通常の開発はVS Code拡張で十分。高度な制御・自動化にはターミナルCLIが有効。
